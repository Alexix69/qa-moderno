Paso crítico

Si el selector es incorrecto o inestable, el test falla — no por un bug real, sino por mala localización.

API de Selenium

findElement() vs findElements()

findElement()

// Devuelve UN solo elemento
WebElement email = driver
    .findElement(By.id("email"));

// ⚠️ Lanza NoSuchElementException
// si no existe
  • Devuelve WebElement
  • Lanza excepción si no existe
  • Uso: un elemento esperado
VS

findElements()

// Devuelve LISTA de elementos
List<WebElement> inputs = driver
    .findElements(By.tagName("input"));

// ✅ Lista vacía si no encuentra
// (no lanza excepción)
  • Devuelve List<WebElement>
  • Lista vacía = sin resultados
  • Uso: múltiples coincidencias
// Ejemplo práctico: Contar inputs en un formulario
List<WebElement> allInputs = driver.findElements(By.cssSelector("#login-form input"));
System.out.println("Campos encontrados: " + allInputs.size()); // → 2

Tipos de localizadores en Selenium

// 8 estrategias de localización disponibles
driver.findElement(By.id("email"));                           // ← ID único
driver.findElement(By.name("username"));                      // ← Atributo name
driver.findElement(By.className("login-button"));             // ← Clase CSS

driver.findElement(By.xpath("//input[@id='email']"));         // ← Expresión XPath

Selectores avanzados

CSS selectors en Selenium

Sintaxis clave

from selenium.webdriver.common.by import By

# ID como CSS
btn = driver.find_element(By.CSS_SELECTOR, "#login")

# Atributo exacto
email = driver.find_element(By.CSS_SELECTOR,
    "input[name='email']")

# Atributo dedicado para testing
cart = driver.find_element(By.CSS_SELECTOR,
    "[data-testid='cart-icon']")

# Prefijo (id dinámico con parte estable)
el = driver.find_element(By.CSS_SELECTOR,
    "input[id^='user_']")

# Contiene
btn = driver.find_element(By.CSS_SELECTOR,
    "button[class*='primary']")

# Sufijo
el = driver.find_element(By.CSS_SELECTOR,
    "button[id$='_login']")

Operadores de atributo

OperadorSignificadoCSS
=Exacto[name='email']
^=Empieza con[id^='user_']
$=Termina con[id$='_login']
*=Contiene[class*='primary']

✅ Ventajas

  • Sintaxis compacta y legible
  • Fácil depurar en DevTools
  • Match parcial: IDs dinámicos

⚠️ Limitaciones

  • Sin parent (no sube)
  • nth-child frágil
  • Sin selección por texto
XPath avanzado

XPath: funciones & ejes

Funciones clave

// contains() → ¿contiene la cadena?
//button[contains(@class,'primary')]

// starts-with() → ¿comienza con...?
//*[starts-with(@id,'user_')]

// normalize-space() → limpia espacios
//*[normalize-space()='Login']

// text() → nodo de texto
//a[text()='Contact']
//button[contains(text(),'Login')]

// Combinaciones poderosas
//button[contains(@class,'login-btn')
  and normalize-space()='Iniciar sesión']

Ejes (Axes) para navegar

// parent → ir al padre
//*[@id='username']/parent::div

// ancestor → subir al ancestro
//*[@id='username']/ancestor::form

// following-sibling → hermano siguiente
//label[normalize-space()='Username']
  /following-sibling::input

// following → siguiente en el DOM
//label[normalize-space()='Username']
  /following::input[1]

Cuándo usar XPath

  • Label → Input sin IDs
  • Tablas: anclar por celda
  • IDs dinámicos + texto estable
  • Navegar relaciones complejas

Sincronización

Elementos dinámicos & esperas

Espera Explícita (recomendada)

# Python
from selenium.webdriver.support.ui import WebDriverWait

wait = WebDriverWait(driver, timeout=10)

# Esperar por condición
wait.until(
    lambda d: d.find_element(
        By.XPATH,
        "//*[normalize-space()='Hello World!']"
    ).is_displayed()
)
// Java
WebDriverWait wait = new WebDriverWait(
    driver, Duration.ofSeconds(10)
);

wait.until(d ->
    d.findElement(By.xpath(
        "//*[normalize-space()='Hello World!']"
    )).isDisplayed()
);

StaleElementReferenceException

¿Qué es?

La referencia al elemento quedó obsoleta: el DOM se re-renderizó y el WebElement apunta a un nodo que ya no existe.

✅ Mitigación

  • Re-localizar justo antes de interactuar
  • No guardar WebElement largo tiempo
  • Esperar staleness + re-find

🚫 Anti-patrones

  • Thread.sleep(5000) → lento y frágil
  • Mezclar waits implícitos + explícitos
  • IDs dinámicos con selector grabado

Buenas prácticas & checklist de selectores

Checklist antes de aprobar PR

  • ☑️ El selector es único — apunta al elemento correcto
  • ☑️ Es estable — no depende de índices o rutas absolutas
  • ☑️ Es legible — se entiende sin inspeccionar el DOM
  • ☑️ Usa anclas semánticas: id, name, data-testid, aria-label
  • ☑️ Está encapsulado en Page Objects
  • ☑️ Tiene sincronización correcta (espera explícita)
  • ☑️ Falla con mensaje claro (timeout bien elegido)

Reglas de oro

Hacer

  • Esperas explícitas con condiciones
  • Selectores cortos y legibles
  • Acordar data-testid con desarrollo
  • Encapsular en Page Objects
  • Verificar unicidad en DevTools

No hacer

  • Thread.sleep() como solución
  • Mezclar waits implícitos + explícitos
  • XPath absoluto /html/body/...
  • nth-child sin justificación
  • By.className con múltiples clases
  • Duplicar selectores en múltiples tests

3. ID Dinámico

# .../dynamic-id
xpath = ("//button["
  "normalize-space()="
  "'Button with Dynamic ID']")

# Click 1

    By.XPATH, xpath).click()

# Refresh → ID cambia, texto NO
driver.refresh()
driver.find_element(
    By.XPATH, xpath).click()
# ✅ Funciona porque no
